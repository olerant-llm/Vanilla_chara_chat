キャラクター設定

性格や口調を自由に設定できます。設定は次の発言から反映されます。初期感情値は「会話と感情をリセット」または「感情を初期値に戻す」を押したときに適用されます。

背景・口ぐせ・好き嫌い・してはいけないことなどを書くほど、キャラが安定します。

初期感情値（0〜100）

感情は会話に応じて自動で増減します。仕組みは感情パラメータと [[EMO]] マーカーを参照してください。

メモリ設定

直近のやり取りはそのまま送り、古くなった会話はキャラクターの「記憶メモ」へ自動で圧縮します。これにより、長い会話でも名前・呼び方・約束などを覚えていられます。整理は返答後に行うため、通常の返答開始は遅くなりにくい設計です。チャット画面のログは消えず、送信する文脈だけを「記憶＋直近」に絞ります。

おすすめは6往復です。多いほど直前の流れを保ちますが、送信量が増えます。

この往復数を超えたら、古い会話を記憶メモへ圧縮します。直近保持より大きい値にしてください。

接続設定

フォルダ内の start-local-server.bat をダブルクリックして開き直してください。
入力した接続情報は、このPCのブラウザ内にだけ保存されます。共有PCでは利用後に設定を削除してください。
なにこれ？
「おてもとLLM」は、ブラウザからそのまま繋げる定額タイプのLLMサービスです。詳細 →
⚠️ ブラウザから直接は繋がらないことがあります（CORS）。 OpenAI・Geminiの公式APIは、ブラウザから直接呼び出すと多くの場合ブラウザ側の制約（CORS）でブロックされます。利用するには中継サーバー（プロキシ）を別途立てるか、CORSを許可したローカル/カスタムのエンドポイントを「カスタム」で指定してください。
また、これらのAPIは利用量に応じて従量課金されることがあります。料金体系をご確認の上、ご自身の判断でご利用ください。

APIサーバーURLを入力します。末尾に /v1/chat/completions が無い場合は自動で補完します。

公開鍵またはAPIキー（パブリック）。認証に使用します。

秘密鍵またはシークレット。リクエストの署名に使用され、サーバー側で検証されます。詳細はサーバー側の仕様を確認してください。

ローカルサーバー（Ollama, LM Studio等）またはOpenAI互換のエンドポイントを指定します。

生成パラメータ

空欄の項目は送信せず、各サービスまたはサーバー側の既定値を使います。

用語集

APIサーバーURL

このチャットがリクエストを送信するサーバーのアドレスです。例：https://api.example.com/v1/chat/completions。通常は OpenAI 互換の形式で、/v1/chat/completions をエンドポイントとして使います。

API キー

サーバーに対して「あなたは誰か」を証明するための認証情報です。秘密鍵（シークレット）として扱い、他人に共有してはいけません。テキスト形式で保存され、リクエストヘッダとして送信されます。

署名認証（HMAC）

APIキーとリクエストの内容から計算されるハッシュ値で、「このリクエストが改ざんされていない」ことを証明します。サーバー側で署名を検証して、正当なリクエストのみを受け入れます。おてもとLLM では HMAC-SHA256( X-Timestamp(秒) + リクエストボディ ) を hex で送信します。

感情パラメータと [[EMO]] マーカー

キャラクターの気分を 0〜100 の数値で表したものです。AIには「まずセリフを書き、最後に必ず [[EMO]]{...} の形で感情の増減量を書く」よう指示し、画面側ではマーカー以降を吹き出しから取り除いて感情の更新にだけ使います。これにより、数値が会話に混ざらず、次のターンの履歴にも残りません。

usage（使用量）

APIの回答に含まれる、このリクエストで消費したトークン数の情報です。prompt_tokens（入力）と completion_tokens（出力）に分かれており、従量課金型のAPIでは課金根拠となります。

stream_options.include_usage

ストリーム形式（リアルタイム逐次表示）でAPIから回答を受け取る際に、同時に使用量（usage）情報も含めるオプション。対応していないサーバーでは無視される場合があります。

自動要約（メモリ圧縮）

会話が増えすぎたとき、古い会話を短い文章（記憶メモ）に圧縮して、新しい会話とセットで送信する仕組みです。これにより、長い会話でもAPIの送信量を抑えながら、名前・呼び方・約束などの文脈を保つことができます。

JSON 保存機能

このチャットの会話履歴と感情の状態を JSON 形式で保存・復元するツール。ブラウザのローカルストレージだけでなく、ファイルとして他のPCやツールで読み込むことができます。
